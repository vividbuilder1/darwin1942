# Darwin 1942 Design System — Sync Notes

## Setup notes

- **Plain JS repo, no `.d.ts` in source**: converter's `exportedNames()` found 0 PascalCase symbols because there are no TypeScript files. Fix: hand-authored `dist/index.d.ts` with all 7 component interfaces + `"types": "./dist/index.d.ts"` in package.json. Carry forward this file; `emptyOutDir: false` in vite.config.js prevents Vite from deleting it on rebuild. [GENERAL]

- **White-text components invisible in preview**: all components use white/ivory text designed for dark backgrounds. The decorator in `.storybook/preview.jsx` wraps every story in `<div style={{background: '#000000'}}>`. Without this, white text is invisible on the white preview canvas. The decorator is auto-bundled into `_vendor/preview-decorators.js`. Confirmed working. [GENERAL]

- **`dist/index.d.ts` deleted by Vite builds**: set `build.emptyOutDir: false` in `vite.config.js`. This is required to preserve the hand-authored declarations file on every build.

- **Button BothCTAs `[GRID_OVERFLOW]`**: the BothCTAs story renders both buttons side-by-side, which is wider than a grid cell. Fixed with `cfg.overrides.Button.cardMode: "column"`.

- **Local image assets not served in preview capture**: `compare.mjs` serves ds-bundle at a local port; image paths `/assets/logo-white.png`, `/assets/logo-dark.png`, and `/assets/hero.jpg` resolve relative to that server root. Fix: copy `assets/` directory to `ds-bundle/assets/` before running compare. Do this before every compare run if assets change. Command: `cp -r assets ds-bundle/assets`.

## Re-sync risks

- `dist/index.d.ts` is hand-authored. If components are added or props change, this file must be updated manually — the build does NOT regenerate it (vite-plugin-dts is installed but produces no output for a plain JS project without TypeScript sources).
- `ds-bundle/assets/` is not generated by the build; it must be copied from `assets/` before running compare. This is not automated.
- The Google Fonts @import in `globals.css` is a CDN fetch. If running in a network-restricted environment, fonts may fall back to system sans-serif.
- All component previews are graded `close` (not `match`) due to clamp-based viewport-responsive sizing — preview captures at a wider viewport than the storybook canvas. This is inherent to the design and not a component defect.
- `ArchiveCaption` appears invisible in its isolated story (white semi-transparent text on black decorator background). It is only visually meaningful when composited inside `HeroSection` over the archival photo. Graded `close` — both sides match (both invisible).
